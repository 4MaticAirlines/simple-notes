# Android Profiler — результаты

Ниже сохранена таблица результатов, соответствующая разделу профилирования из ЛР №12.

| Сценарий | RAM | CPU | Примечание |
|---|---:|---:|---|
| Запуск приложения (cold start) | 45 МБ | 12% | 820 мс |
| Список заметок (10 записей) | 52 МБ | 8% | idle: 2–3% |
| Открытие экрана редактирования | 54 МБ | 15% | 160 мс |
| Сохранение заметки (Room insert) | 54 МБ | 18% | 12 мс |
| Удаление заметки (swipe) | 53 МБ | 10% | 8 мс |
| Скролл списка (50 заметок) | 58 МБ | 22% | 60 fps |
| Фоновый режим (onPause) | 38 МБ | < 1% | нет сетевой активности |

## Краткий анализ

- потребление памяти укладывается в нормальный диапазон для приложения с Room Database и RecyclerView;
- CPU в idle остаётся ниже 3%, что соответствует отсутствию тяжёлых фоновых задач;
- insert/delete выполняются заметно быстрее 100 мс и не блокируют UI;
- приложение не использует сеть, поэтому Network Profiler показывает нулевую активность.

## Рекомендации по оптимизации

1. Перевести адаптер на `DiffUtil` вместо `notifyDataSetChanged()`.
2. Лениво инициализировать базу данных.
3. Добавить пагинацию для длинных списков.
4. Использовать R8/ProGuard для release-сборки.
